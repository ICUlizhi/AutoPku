更新日志

AutoPku 从单体脚本到模块化 Agent Team 平台的演进历程。

阶段一 2024 年初
原型搭建
单体 skill.md 脚本，实现基础的通知获取与作业下载功能，验证自动化可行性。
阶段二 2024 年初
Agent Team 化重构
引入多 Agent 协作模式，单课程内拆分 Coordinator、Parser、Solver 等角色，任务处理效率显著提升。
阶段三 2024 年中
完善项目介绍
添加系统截图与使用说明，建立基础文档体系，降低新用户上手门槛。
阶段四 2024 年中
跨平台支持
支持 Claude Code Agent Teams 与 OpenAI Codex 双模式运行，引入运行时检测机制。
阶段五 2024 年末
pku3b 工具链升级 v0.11.0
升级底层教学网 CLI 工具，新增公告获取与课表查询能力，扩展系统覆盖场景。
阶段六 2024 年末
PDF 处理能力补强
引入 PyMuPDF 与 pdfplumber 双引擎，提升复杂排版 PDF 的解析准确率。
阶段七 2025 年初
架构大重构 — 模块化 Sub-skills
将单体 skill 拆分为 runtime / tools / tasks 三层 sub-skills，主入口只负责任务路由，系统可扩展性大幅提升。
阶段八 2025 年初
笔记撰写任务
新增 write-notes.md 任务，支持从课件 PDF 提取数学核心内容并生成结构化的课程笔记。
阶段九 2025 年中
自然语言驱动简化
优化意图识别与路由逻辑，用户无需记忆命令关键词，直接以自然语言描述需求即可触发对应任务。
阶段十 2025 年中
Kimi Code CLI 支持
新增 Kimi Code CLI Agent Team 语法适配，完成 Claude / Codex / Kimi 三大主流平台全覆盖。
阶段十一 2025 年末
PDF 渲染踩坑与 Callout 修复
修复 Markdown 转 PDF 过程中的 LaTeX 公式渲染与 callout 样式兼容性问题，输出质量达到生产可用标准。

当前系统能力汇总

维度 能力
核心任务 同步通知、完成作业、撰写笔记（3 大任务）
AI 运行时 Claude Code、OpenAI Codex、Kimi Code CLI（3 大平台）
输出格式 Markdown、PDF（含 LaTeX 公式）、教学网直接提交
架构模式 主入口路由 + 模块化 Sub-skills + Agent Team 协作
