准备工作

让 AI 帮你搞定一切前置步骤

嗨！如果你是第一次接触代码，别担心——你不需要记住任何命令，也不需要理解 Git 的底层原理。本页面的核心理念是：让 AI 成为你的助手，你只需要用自然语言告诉它想做什么。

💡 小提示

下面每一步都附带了"对 AI 说的话"，你可以直接复制发给 Kimi、Claude 或其他 AI 助手。AI 会帮你执行具体操作，你只需要跟着确认即可。

五步完成环境准备

1

注册 GitHub 账号

如果你还没有 GitHub 账号，去 github.com 注册一个（完全免费）。

或者，直接对 AI 说：

帮我注册一个 GitHub 账号

AI 会引导你填写用户名、邮箱、密码，并验证邮箱。记住你注册的用户名，后面会用到。

2

Fork 仓库（让 AI 帮你）

Fork 听起来很技术，其实就是把 AutoPku 仓库复制一份到你自己的账号下，这样你就可以自由修改了。

对 AI 说：

帮我 Fork 这个仓库 https://github.com/ICUlizhi/AutoPku

AI 会帮你完成 Fork，或者给你一步步的截图指引。完成后，你的仓库地址会是这样：

URL 
https://github.com/你的用户名/AutoPku
3

Clone 到本地

Clone 就是把 GitHub 上的仓库下载到你的电脑里。对 AI 说：

帮我把 Fork 的 AutoPku 仓库克隆到本地

AI 会自动执行命令，把代码下载到你电脑的某个文件夹里。它执行的命令大概长这样（你不需要自己敲）：

BASH 
git clone https://github.com/你的用户名/AutoPku.git
4

创建你的分支

对 AI 说：

帮我创建一个 feature 分支，我要做某某功能

AI 会执行类似下面的命令，为你创建一个独立的工作空间：

BASH 
cd AutoPku
git checkout -b feature/你的功能名称

为什么要新建分支，而不是直接在 main 上改？想象一下，main 分支是大家一起用的"主干道"，如果你直接在上面施工，可能会和别人发生冲突。feature 分支就是给你单独修的一条"施工便道"，想怎么改就怎么改，改完再合并回去。

5

配置 Git 身份

这一步是告诉 Git"你是谁"，这样你提交的代码才能署上你的名字。对 AI 说：

帮我配置 Git 用户名和邮箱

AI 会设置成你在 GitHub 上注册的用户名和邮箱，执行的命令类似：

BASH 
git config user.name "你的GitHub用户名"
git config user.email "你的GitHub邮箱"

检查清单

✅ 准备工作完成后，你应该有：
  • GitHub 账号
  • Fork 了 AutoPku 仓库
  • 本地有仓库副本
  • 创建了自己的 feature 分支
  • Git 身份已配置

全部打勾了？太棒了！你已经完成了所有前置准备，可以正式开始写代码了。
