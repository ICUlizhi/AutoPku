给北大纯小白的 AutoPku 使用说明

> 这篇文档假设你：
> - 不会写代码
> - 没用过 Kimi Code / Claude Code
> - 只想：**“我给 AI 一些账号信息，它自己帮我管教学网和作业。”**

## 一、AutoPku 到底是干嘛的？

一句话总结：
**你只需要给出学号密码 + 自然语言指令，AI 就可以帮你从教学网同步通知、整理课程信息，甚至完成并提交作业。**

更具体一点，它能做的包括（根据 README）：

- 自动登录教学网（用 pku3b 等工具）
- 获取所有课程的通知、作业、附件
- 为每门课创建一个独立的 Agent，分头处理
- 解析 PDF 作业 → 生成答案 → 排版成 Markdown / PDF
- 把 PDF 答案自动提交到教学网
- 生成一个「通知和作业的汇总文件」

重点是：**你不需要自己部署一套复杂系统**，而是把这个仓库作为一个“Skill”交给 Kimi Code / Claude Code / Codex 这样的 AI 编程助手去执行。

***

## 二、先搞清楚：Kimi / Kimi Code 是什么？

### 2.1 Kimi 是什么？

Kimi 是一个中文友好的大模型 AI 助手，你可以在网页端、App、小程序里和它聊天、让它写作业、看论文、写代码等。

你可以通过这些方式打开 Kimi：

- 浏览器搜索 “Kimi 智能助手”，打开官网即可使用
- iOS：在 App Store 搜索 “Kimi 智能助手”
- 安卓：各大应用商店搜索 “Kimi 智能助手”
- 微信：搜索“小程序：Kimi 智能助手”

> 这些入口主要用于聊天、看文档之类。
> **AutoPku 推荐你使用的是专门的编程版：Kimi Code。**

### 2.2 Kimi Code 又是什么？

Kimi Code 是 Kimi 官方的 **AI 编程助手 / Coding Agent**，可以在终端或编辑器里理解整个项目、读写文件、执行命令，是一个“会自己动手做事的 AI 开发搭子”。

对本项目来说，它的作用是：

- 帮你下载 `AutoPku` 仓库
- 自动安装依赖、配置环境
- 调用 `skill.md` / `SKILL.md` 里的逻辑，接管和教学网相关的一切操作

你可以认为：**Kimi Code 是“帮你跑 AutoPku 的人”，AutoPku 是“帮你搞定教学网和作业的工具箱”。**

***

## 三、我需要准备什么？

在开始之前，你需要：

- 一台能上网的电脑（Windows / macOS 都可以）
- 一个可用的 Kimi 账号（手机号 / 邮箱注册即可）
- 一点点耐心，照着文档一步一步来

如果你用不了 Kimi Code，也可以用 README 里提到的 Claude Code（cc）或 Codex，它们的定位也都是可以读仓库、执行命令的“AI 编程助手”。

***

## 四、如何打开 Kimi Code？

Kimi Code 有两种常见形态：

1. **Kimi Code CLI（终端版）**：在命令行里运行 `kimi` 命令，和它聊天、让它执行任务。
2. **Kimi Code 结合编辑器（VS Code / 其他 IDE 插件）**：在编辑器里出现一个“AI 面板”，能读你打开的项目文件。

对纯小白来说，推荐两种路径：

### 4.1 如果你身边有会折腾终端的同学

你可以让对方帮你完成这几步（来自 Kimi Code 官方和社区教程）：

1. 在电脑上安装 Python 环境（通常就装 3.x 即可）。
2. 安装 `uv` 或其他 Python 包管理工具。示例（只给会折腾的同学看）：

```bash
pip install uv
```
```

3. 安装 Kimi Code CLI：

```bash
uv tool install --python 3.13 kimi-cli
``` 
```

4. 安装好以后，在终端输入：

```bash
kimi --version
```

有版本号说明安装成功。
5. 在终端输入：

```bash
kimi
```

按提示用浏览器登录你的 Kimi 账号完成授权。

> 上面这 1–5 步只需要做一次。以后每次用，只要打开终端输入 `kimi` 就行。

完成后，你在终端里看到一个类似聊天界面的界面，就可以进入第五步“让它下载并执行 AutoPku”。

### 4.2 如果你完全不想碰命令行

那你可以考虑以下方案之一：

- 使用 **Claude Code（cc）**：它是 Anthropic 出的 AI 编程助手，同样擅长读仓库、跑命令。
- 使用 **Codex 或其他支持“读项目 + 执行命令”的 AI 工具**：很多国产编辑器也集成了这样的能力。

README 里明确写了，本项目保留了适配 Claude Code 和 Codex 的入口文件（`skill.md` 和 `codex/autopku/SKILL.md`）。

如果你身边有已经在用这些工具的同学，**最简单的方法是：让他们在自己的工具里跑 AutoPku，你只负责提供教学网账号和需求文本即可。**

## 五、让 Kimi Code 帮你跑 AutoPku

这一节是假设你已经能打开 Kimi Code（终端里输入 `kimi`）或打开一个类似的 AI 编程助手界面。

### 5.1 把这句话复制给 Kimi Code

进入 Kimi Code 对话界面后，不要自己先乱敲命令，直接复制下面这一句发给它：

```text
下载 https://github.com/ICUlizhi/AutoPku，并执行这个skill
```

这句提示来自项目 README，是官方推荐的第一步。

Kimi Code 会自动做这些事（大意，具体实现由 Agent 决定）：

- 下载 `AutoPku` 仓库
- 识别当前的工作目录
- 理解 `skill.md` / `SKILL.md` 里的逻辑
- 配置运行环境（包括安装 `pku3b` 等工具）

对你来说，这一步的重点是：**你不需要自己执行 `git clone`、不需要自己安装依赖，只要给出这一句让它“执行这个 skill”。**

### 5.2 按提示提供学号密码等信息

当它准备好之后，一般会提示你需要提供：

- 北大教学网的学号
- 对应密码（注意安全，尽量只在你信任的本地环境里执行）
- 有时还会问你要把结果输出到哪里（比如某个目录）

你只要照实回答即可。
这些信息是它自动登录教学网、获取课程和作业所必需的。

## 六、怎么给它下指令？

当它提示“已经配置好 AutoPku，可以开始任务”之后，你就可以用**自然语言**给任务了。

README 给了三个典型的例子：

```text
帮我同步这周的通知
写一下操作系统实验班的笔记
量子力学的第五次作业还没做，帮我搞定
```

你也可以举一反三，比如：

- 「帮我列出本学期所有课程还有哪些作业没交」
- 「把这周所有有截止日期的作业帮我汇总成一个 markdown」
- 「帮我完成数值分析课的第 3 次作业，并生成可提交的 PDF」

它会根据 Skill 里的设计，自动完成后续一长串操作。

***

## 七、背后它在做什么？（大概知道就行）

你不需要看懂代码，但知道大概流程，会让你更放心。

根据 README 描述，当你只提供学号密码，让它“同步通知”时，它会做这样的事情：

1. **安装并登录 pku3b**：下载适配版本，用脚本登录教学网。
2. **获取课程数据**：拉取所有课程、作业、通知列表。
3. **召唤 Agent Team**：
    - 为每门课创建一个独立的 Agent（并行执行）
    - 每个 Agent 下载附件、生成课程通知摘要
4. **生成汇总报告**：
    - 输出一个《通知摘要汇总.md》作为所有课程的总览文件

若你指定某门课、某次作业，它会进一步执行一个 5 阶段流程：

1. PDF 解析（从作业 PDF 中抽取题目）
2. 逐题解答（可能会参考资料）
3. 生成 Markdown 答案文件
4. 渲染成 PDF
5. 提交到教学网

生成的目录结构大致类似：

```text
test/
├── 通知摘要汇总.md
├── 简明量子力学/
│   ├── 作业/
│   ├── 通知/
│   ├── 资料/
│   └── 通知摘要.md
└── ...
```

和某次作业对应的目录则类似：

```text
test/简明量子力学/C
├── 作业/
│   ├── Homework202605.pdf           # 原始作业
│   └── Homework202605_answer.md     # 答案 Markdown
├── 提交/
│   └── Homework202605_answer.pdf    # 最终 PDF（已提交）
├── homework_parsed.json             # PDF 解析结果
└── answers.json                     # 答案数据
```

这些文件都在本地，你可以自己打开检查。

## 八、特别重要：作业提交一定有“人类确认”

为了避免“AI 自己帮你乱交作业”这样的灾难，项目专门设计了明确的用户确认流程：

1. 它会先列出这门课当前所有“待交作业”。
2. 你亲自选择要完成哪一次（比如 “第 5 次作业”）。
3. 它会再二次确认一次。
4. 只有你确认后，它才会真正开始解题、生成 PDF、提交。
5. 过程中你随时可以让它停下。

此外，还有一条硬性规则：
**禁止自动选择“最新作业”来完成，必须是你明确选择的那一个。**

所以，在它说“我要提交 XXX 作业”时，请务必：

- 仔细确认课程名
- 仔细确认作业名称 / 编号
- 不确定就让它“只做到生成答案文件，不要提交”，自己看一眼再决定

## 九、遇到问题怎么办？

就算有了“小白文档”，还是可能遇到奇怪的问题，比如：

- Kimi Code 安装不成功
- Kimi Code 不认识某个命令
- AutoPku 跑了一半报错
- 教学网改版导致逻辑失效

这个时候，最有用的不是一句“挂了”，而是准备好下面这些信息：

1. 你用的是哪一个工具：
    - Kimi Code CLI？
    - Claude Code（cc）？
    - 还是某个别的 AI 编程助手？
2. 你发给它的**原始提示词**（尤其是那句“下载仓库并执行 skill”）。
3. 报错信息的截图（终端 / 对话框）。
4. 简单说明你希望它做什么（同步通知还是完成某门课作业）。

然后你可以去 AutoPku 仓库提 issue：
[https://github.com/ICUlizhi/AutoPku/issues](https://github.com/ICUlizhi/AutoPku/issues)

越具体，越容易有人帮你解决。
